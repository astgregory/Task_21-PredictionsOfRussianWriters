# Определение схожести стиля текста с русскими писателями

Проект реализует модель машинного обучения для определения, на какого из русских писателей похож заданный текст. Решение основано на анализе стиля написания с использованием нейронных сетей и предобученных эмбеддингов.


## Задача

Определить, на какого русского писателя (из представленного набора) похож заданный текст, с достижением:
* средней точности более 70 % на тестовых образцах (3 балла);
* точности более 90 % (дополнительный +1 балл);
* предсказания для собственного текста или фрагмента произведения (дополнительный +1 балл).


## Функциональность

Модель выполняет следующие задачи:
1. Загрузка и распаковка датасета с произведениями русских писателей.
2. Токенизация текстов.
3. Балансировка датасета по количеству слов у разных авторов.
4. Разбиение данных на обучающую, валидационную и тестовую выборки.
5. Обучение модели с замороженным слоем эмбеддинга (на весах Natasha).
6. Построение матрицы ошибок на тестовой выборке.
7. Предсказание автора для нового текста и визуализация вероятностей в виде круговой диаграммы.

## Используемые технологии

* **Keras/TensorFlow** — построение и обучение нейронной сети.
* **Navec (Natasha)** — предобученные эмбеддинги для русского языка.
* **Scikit‑learn** — матрица ошибок.
* **Matplotlib** — визуализация графиков и диаграмм.
* **Python** — основной язык программирования.

## Архитектура модели

Комбинированная архитектура включает:
* Слой эмбеддинга с предобученными весами Natasha (заморожен).
* Двунаправленные LSTM‑слои (2 × Bidirectional(LSTM(8))).
* GRU‑слои (2 × GRU(16)).
* Полносвязные слои с Dropout и BatchNormalization.
* Выходной слой с softmax для классификации по авторам.

## Данные

    - Датасет произведений русских писателей (проза и поэзия) в формате TXT, организованный по папкам авторов.

## Результаты

    - Точность на тестовой выборке: более 97 % (достигнуто в ходе обучения).

    - Матрица ошибок: визуализирована для анализа качества классификации.

    - Круговая диаграмма: показывает вероятности принадлежности текста к каждому автору.

## Вклад

    - Реализована балансировка датасета для устранения дисбаланса между авторами.

    - Использованы предобученные эмбеддинги Natasha для улучшения качества векторизации.

    - Комбинированная архитектура (LSTM + GRU) показала высокую точность классификации.

    - Визуализация процесса обучения и результатов (графики точности/потери, матрица ошибок, круговая диаграмма).
